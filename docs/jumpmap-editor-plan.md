# 점프맵 + 맵 에디터 개발 계획 (확정본)

이 문서는 점프맵과 맵 에디터의 **확정된 요구사항/규칙/포맷**을 한 곳에 모아둔 기준 문서입니다.
개발/수정 시 반드시 이 문서를 기준으로 일관성을 유지합니다.

참조 지침: `docs/jumpmap-editor-guidelines.md`

---

## 1) 기본 원칙
- 맵 에디터 화면은 **실제 게임 화면과 동일**해야 한다.
- 에디터 기능은 화면을 방해하지 않도록 **접기/펼치기** 가능해야 한다.
- 비율은 절대 변형되지 않는다(가로:세로 고정). 화면 크기에 따라 스케일만 변한다.
- **투명 영역은 빈 공간**이다. PNG 전체를 충돌로 취급하지 않는다.
- **히트박스는 반드시 별도 지정**한다.

---

## 2) 화면/카메라 규칙
- 기준 비율: **16:9**
- 자동 스케일: 화면 크기에 맞게 전체 스케일 조정
- 캐릭터 위치: **화면 중앙보다 약간 아래**
- 추락 시 카메라: **부드럽게 추적**

---

## 3) 맵 크기 규칙
- 기본 맵 크기: **2400 × 12000**
- 에디터에서 **맵 크기 확장 가능**
  - 버튼 확장: 가로 +800, 세로 +3000
  - 숫자 입력 확장: 가로/세로 직접 입력

---

## 4) 시작 지점 규칙
- 에디터에서 **시작 지점 지정 모드** 제공
- 시작 지점은 플레이어 스폰 기준(발바닥 위치 기준)
- 세이브포인트는 시작 지점과 별개로 여러 개 저장 가능(이름 포함)
- 기본 진입 시 시작 지점으로 자동 스크롤
- 빠른 이동에서 “시작 지점” 버튼 제공
- 기본 진입 시 **맨 아래층 발판 + 시작 지점**을 자동 배치

---

## 5) 발판/오브젝트 배치 규칙
- PNG 투명 영역 = 빈 공간
- 히트박스는 별도 지정(기본 사각형 + 단일 다각형 지원)
- 히트박스 여러 개 허용
- 히트박스 개별 회전(0~359도) 지원
- 히트박스 편집은 **드래그 + 숫자 입력** 모두 제공
- 오브젝트/히트박스/플레이어는 선택 대상을 분리하여 편집한다
- 질감 오브젝트는 **한지/돌/얼음/단일 색상** 4종만 사용한다
- 단일 색상 질감은 색상값(`textureColor`)을 저장/불러오기에서 보존한다

---

## 6) 스케일/회전 규칙
- 기본 스케일: **0.55**
- 스케일 범위: **0.2 ~ 2.0**
- 스케일 스텝: **0.05**
- 이미지별 스케일 저장
- 회전: **자유 회전 (0~359)**
- 보조 스냅: **Shift 드래그 시 15도 단위**
- 좌우 반전 / 상하 반전 지원

> 회전 히트박스는 **로컬 좌표계 기준 OBB 충돌**로 반영됨.

---

## 7) 격자(보조선) 규칙
- 격자는 **보조선**이며 플레이에 영향 없음
- 스냅 ON/OFF 가능
- 격자 크기: **8 / 16 / 32 / 64** 전환
- 기본값: **32px**

---

## 8) 캐릭터 규칙
- 캐릭터 히트박스는 **폭/높이/발 간격** 직접 조정 가능
- 캐릭터 크기(스케일) 조정 가능, 모든 애니메이션 프레임에 동일 적용
- 캐릭터 자르기(크롭) 지원
- 값은 저장 포맷에 포함되어 재사용 가능

---

## 9) 테스트 모드 규칙
- 에디터에서 바로 테스트 실행
- 방향키 + 점프키 지원
- 2~6인 분할 테스트 지원
- 테스트 모드에서 세이브포인트를 선택해 즉시 해당 지점으로 이동(워프) 가능
- 추락 시 바닥/발판을 만날 때까지 계속 하강
- 클릭 텔레포트 입력은 사용하지 않는다

---

## 10) 다인 플레이 규칙
- 2~6인 모두 지원
- 분할 화면 내부에서만 조작 가능
- 각 플레이어는 **독립 인스턴스**
- 맵 데이터는 공유, 상태는 분리

---

## 11) 높이 기록 시스템
- 기록 모드 2개 제공
  - 종료 순간 높이
  - 최고 높이
- 기본값: **최고 높이 모드**

---

## 12) 저장 포맷 (기준)
```json
{
  "version": 2,
  "schema": "jumpmap-editor-map",
  "savedAt": "2026-02-07T00:00:00.000Z",
  "mapSize": { "w": 2400, "h": 12000 },
  "grid": { "size": 32, "snap": true },
  "camera": { "yBias": 0.46, "smooth": 0.18 },
  "background": { "color": "#ffffff", "image": "", "texture": "" },
  "playerHitbox": { "width": 80, "height": 120, "footInset": 8 },
  "playerHitboxOffset": { "x": 0, "y": 0 },
  "playerScale": 1,
  "playerCrop": null,
  "physics": {
    "fallSpeed": 600,
    "jumpSpeed": 1000,
    "jumpHeight": 240,
    "moveSpeed": 220
  },
  "startPoint": { "x": 200, "y": 11800 },
  "savePoints": [],
  "editorOptions": {
    "autoBasePlatform": true,
    "autoScrollStart": true,
    "autoSelectAfterPlace": true
  },
  "spriteProfiles": {},
  "objectGroupPresets": {},
  "hitboxPresets": {},
  "objects": [],
  "spriteDefaults": {}
}
```

- 기본 저장 키는 `spriteProfiles`이며, `hitboxPresets/spriteDefaults`는 레거시 호환용으로 유지한다.
- `objectGroupPresets`는 "묶은 오브젝트 프리셋" 저장/불러오기용 정식 필드로 사용한다.
- 로드 시에는 레거시 포맷(`v0`, `v1`)을 자동 마이그레이션한 뒤 정규화한다.
- 상위 버전 파일(`v2` 초과)은 가능한 필드만 로드하고 경고를 남긴다.

---

## 13) 에디터 UI 레이아웃
- 좌측: 발판 팔레트 + 프로파일 확정 팔레트
- 중앙: 맵 캔버스, 미니맵, 커스텀 스크롤바
- 우측: 속성/선택대상/히트박스 편집
- 상단: 저장/불러오기/초기화/로컬슬롯(5개, 슬롯명 지정/수정)/테스트/되돌리기
- 중앙 상단 툴바: 빠른 이동(상단/중단/하단/시작지점), 맵/그리드/배경/물리
- 정밀 작업판: 선택 오브젝트 1개를 확대(픽셀 단위)해 크롭/히트박스를 미세 조정하는 오버레이 편집창

---

## 14) 에셋 관리 규칙
- 원본 에셋 폴더는 손상시키지 않는다.
- 필요 시 파생 에셋 폴더를 별도로 두고 trim/crop 결과를 저장한다.
- 이미지 교체 후에는 에디터에서 즉시 점검한다.

---

## 15) 분리/세부화 개발 계획 (신규)

### 15.1 단계 운영 원칙
- 각 단계는 "작은 변경 + 즉시 검증"으로 진행한다.
- 단계 완료 전 다음 단계로 넘어가지 않는다.
- 단계별 문서(`status`)를 반드시 갱신한다.

### 15.2 단계별 실행
1. **Phase 0: 기준선 고정**
- 목표: 현재 동작 스냅샷과 검증 체크리스트 확정
- 산출물: status 갱신, 체크리스트

2. **Phase 1: 순수 유틸 분리 (무동작 변경)**
- 대상: 좌표/회전/충돌 보조함수, 스냅/클램프 함수
- 금지: 이벤트/렌더 로직 변경

3. **Phase 2: 저장/불러오기 안정화**
- 대상: map io 모듈화, schema 검증/예외 처리
- 결과: 손상 JSON 로드 안전 처리

4. **Phase 3: 테스트 런타임 분리**
- 대상: test loop, 입력 처리, 플레이어 물리
- 결과: 에디터 로직과 테스트 로직 결합도 축소

5. **Phase 4: 렌더 경량화**
- 대상: 드래그 중 전체 재렌더 최소화
- 결과: 오브젝트 수 증가 시 편집 반응 개선

6. **Phase 5: 에셋 정규화 + 프리셋 워크플로우**
- 대상: 투명 여백 최소화(trim), sprite profile, 확정 팔레트
- 결과: 배치 즉시 히트박스/크롭 적용

7. **Phase 6: 점프맵 완성 안정화**
- 대상: 다인 테스트, 저장 호환, 문서 동기화
- 결과: 실제 제작 가능한 운영 상태
- 검증: `node scripts/jumpmap-phase6-validate.mjs` + `docs/jumpmap-editor-phase6-checklist.md`

### 15.3 단계별 공통 검증
- 배치/선택/이동/크기/자르기(오브젝트/히트박스/캐릭터)
- 시작지점 이동과 테스트 스폰
- 저장/불러오기 후 동일성
- 테스트 모드 이동/점프/착지/카메라

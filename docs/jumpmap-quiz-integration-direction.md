# 점프맵-퀴즈코어 통합 방향 메모 (초안)

작성일: 2026-02-21  
범위: `jumpmap-editor` 테스트/런타임과 `quiz core` 연결 방향  
목적: 현재까지 논의한 아이디어를 정리해 두고, 이후 구현/수정 시 참고 메모로 사용

상태: `초안(검토 중)`  
주의: 이 문서는 확정 사양이 아니라 방향성 메모이며, 구현 과정에서 조정될 수 있음

---

## 1. 현재 방향(초안)
- 게이지 시스템은 `퀴즈코어`가 아니라 `점프맵 런타임`이 소유한다.
- 퀴즈코어는 문제 출제/채점/점수 계산만 담당한다.
- 두 시스템 연결은 `integration bridge`로만 수행한다.
- 로컬 우선 개발을 유지한다(서버/DB 필수 아님).

이 구조를 쓰면 점프맵은 점프맵대로 물리/게이지를 안정화하고, 퀴즈코어는 다른 게임 모드에도 재사용 가능하다.

---

## 2. 게이지 규칙 (논의안)
- 지상 이동: 게이지 소모
- 지상 점프: 게이지 소모
- 공중 이동: 게이지 0이어도 허용
- 더블점프: 게이지 체크 필요, 게이지 0이면 불가
- 게이지 부족 시: 이동/점프 차단 규칙은 `지상 기준`으로 우선 적용
- 퀴즈 정답 시: 게이지 회복
- 퀴즈 오답 시: 회복 없음(또는 낮은 회복량은 추후 설정값으로 열어둘 수 있음)

주의:
- 공중 상태에서 완전 정지/고착이 생기지 않도록, 입력 처리와 게이지 차단 로직을 분리한다.
- 차단은 물리 이전 단계에서 일관되게 적용한다.

---

## 3. 모듈 경계

### 3.1 Jump Runtime (점프맵 런타임)
- 물리 처리(이동/점프/충돌/경사/미끄러짐)
- 게이지 소모/회복 상태
- 입력 허용/차단 판정
- 테스트모드 루프

### 3.2 Quiz Core
- 문제은행 구성
- 문제 출제
- 정답 판정
- 점수/콤보/시간 계산

### 3.3 Integration Bridge
- 런타임 이벤트 전달
- 퀴즈 요청/응답 중계
- 게이지 변경 이벤트 전달
- 런타임 일시정지/재개 트리거

---

## 4. 런타임-퀴즈 이벤트 계약(초안)

## 4.1 런타임 -> 브리지
- `gauge:consume` `{ reason, amount, playerId }`
- `gauge:empty` `{ reason, playerId }`
- `quiz:request` `{ reason, playerId, context }`

## 4.2 브리지 -> 런타임
- `quiz:resolved` `{ accepted, correct, refillAmount, playerId }`
- `gauge:refill` `{ amount, playerId }`
- `runtime:pause` / `runtime:resume`

원칙:
- 런타임이 직접 퀴즈 엔진 세부 구현을 알면 안 된다.
- 브리지가 퀴즈코어와 런타임 사이의 번역층 역할을 한다.

---

## 5. 맵 에디터 테스트모드 통합 전략
- `테스트모드 OFF`: 기존 물리 테스트만
- `테스트모드 MOCK`: 정답/오답 버튼으로 게이지/흐름 확인
- `테스트모드 LIVE`: 실제 퀴즈코어 호출

장점:
- 맵 편집 중 즉시 게임 루프(이동 -> 게이지 소모 -> 퀴즈 -> 회복) 검증 가능
- 구현 전 단계에서도 브리지 계약 검증 가능

## 8. 구현 우선순위 (초안)
1. 브리지 실연결: 런타임에서 `consumeGauge/requestQuiz/refillGauge` 실제 호출
2. 게이지 차단 규칙 반영: 지상/공중/더블점프 조건 분리
3. 테스트모드 3단계(OFF/MOCK/LIVE) 스위치 추가
4. 퀴즈코어 LIVE 연결 안정화
5. 멀티플레이(2~6인)에서 플레이어별 게이지/퀴즈 흐름 분리
